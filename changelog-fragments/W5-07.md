### W5-07 — 13 additional CoreProtect-parity per-event toggles

Extended `GuardianConfig.Actions` with 13 new boolean toggles matching the
`Config.java` shape shipped by CoreProtect. All are appended at the end of
the record so existing 18-arg positional constructors continue to compile
via a `@Deprecated` backward-compat shim on `Actions`.

New fields (CP defaults in parens):

| field | default | rationale |
| --- | --- | --- |
| `logNaturalBreaks` | `true` | crops trampled, tall grass mob-broken, etc. |
| `logTreeGrowth` | `true` | sapling &rarr; tree placements |
| `logMushroomGrowth` | `true` | huge-mushroom growth |
| `logVineGrowth` | `true` | vine / kelp / cave-vine growth |
| `logSculkSpread` | `true` | sculk-catalyst spread |
| `logPortals` | `true` | nether/end portal formation |
| `logWaterFlow` | `false` | per-tick block change — expensive |
| `logLavaFlow` | `false` | per-tick block change — expensive |
| `logFireExtinguish` | `true` | rain / water / player extinguish |
| `logCampfireStart` | `true` | campfire lighting |
| `logHopperMetaFilter` | `false` | opt-in NBT-aware hopper dedup |
| `logDuplicateSuppression` | `true` | collapse identical adjacent rows |
| `logCancelledChat` | `false` | opt-in: log chat cancelled by other plugins |

`ConfigLoader.migrateForwardCompat()` heuristically backfills the CP-parity
defaults when a pre-W5-07 config comes in with all thirteen toggles absent
(deserialising to `false`). Any single `true` is treated as an opt-in and
the operator's exact values are preserved verbatim.

`GuardianConfig.defaults()` updated to construct the full 31-arg
canonical form. `validate()` unchanged — no new range checks required
(pure booleans).
