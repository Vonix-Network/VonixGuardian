# W4-14 — Ecosystem plugin discoverability

## Added
- **`docs/PLUGINS.md`** — full authoring guide for third-party mods that want
  to plug into VonixGuardian as a soft-dep. Covers the Gradle coordinate
  (`network.vonix.guardian:vonixguardian-core:1.2.0`, `compileOnly`,
  `transitive = false`), reflection-based soft-dep pattern, and worked
  examples for the three primary integration points:
  - subscribing an `EventHook` at the `PRE_LOG` stage,
  - reusing an existing `PermissionNode` for command-side gating,
  - submitting an `ActionRecord` via `EventSubmitter`.
  Blueprint sketches for Dynmap markers, Discord relay (in + out bound), and
  per-region protection / claim override hooks are included.
- **`docs/API.md`** — appended a **Public class index** section listing every
  FQN in the stable v1.2.0 surface (grouped by facade / typed API / query /
  perms / rollback / attribution / config), plus a link to the CI-published
  Javadoc site (`https://vonix-network.github.io/VonixGuardian/javadoc/`)
  and coordinates of the sources+javadoc jars.
- **`core/build.gradle`** — extended the `publishing` block with a second
  `maven` repository target for Maven Central (staging URL
  `https://s01.oss.sonatype.org/service/local/staging/deploy/maven2/`,
  marked `TODO(sonatype)` until the namespace claim clears). Credentials
  read from `ORG_GRADLE_PROJECT_ossrhUsername` /
  `ORG_GRADLE_PROJECT_ossrhPassword` env vars via project-property
  fallback. Enriched the POM with `inceptionYear`, `issueManagement`,
  developer email / org, and `license.distribution = 'repo'` — all Sonatype
  validation requirements. Added `withJavadocJar()` so the sources+javadoc
  pair required by Central are produced automatically.
- **Signing wiring** — new `signing {}` block using `useGpgCmd()` so CI can
  drive gpg-agent via `ORG_GRADLE_PROJECT_signingKeyName` /
  `ORG_GRADLE_PROJECT_signingPassphrase`. Signing is only *required* when
  the task graph contains `publishMavenPublicationToMavenCentralRepository`,
  so day-to-day local installs and the existing GitHub Packages pipeline
  stay unsigned and unchanged.

## Notes
- **No actual publish.** The MavenCentral repo block is config-only and
  awaiting a Sonatype account. A `# TODO(sonatype)` comment marks the URL.
- **GitHub Packages publishing is preserved** exactly as before.
- Build verified with `./gradlew -PbuildProfile=coreonly :core:build`.
