# W4-04 — Fabric mixin infrastructure skeleton

**Type:** infra
**Scope:** fabric (all four MC cells: 1.18.2, 1.19.2, 1.20.1, 1.21.1)

## What

Added per-cell Fabric mixin configuration skeleton so W4-06 can drop
mixin classes into a pre-wired package without touching build files:

- New `mc-<v>/fabric/src/main/resources/vg.mixins.json` in each cell:
  - `package`: `network.vonix.guardian.mc.<v>.fabric.mixin`
  - `minVersion`: `0.8`
  - `refmap`: `vg.refmap.json`
  - `compatibilityLevel`: `JAVA_17` (1.18.2, 1.19.2, 1.20.1) / `JAVA_21` (1.21.1)
  - Empty `mixins` / `server` / `client` arrays (skeleton — no mixins yet)
  - `injectors.defaultRequire = 1`
- `fabric.mod.json` in each cell: added `"mixins": [ "vg.mixins.json" ]`.

Gradle wiring: no changes required — `fabric-loom` (already applied in
each cell's `build.gradle`) handles mixin annotation processing and
refmap generation automatically once the config is present and
referenced from `fabric.mod.json`.

## Why

W4-06 will add the first real mixins (audit hooks). Splitting the
plumbing out here keeps that follow-up PR focused on the mixin classes
themselves and keeps this change strictly additive / no source impact.

## Verification

- `./gradlew -PbuildProfile=coreonly :core:build -x test` — green.
- No `.java` sources modified.
