# W4-12 — i18n foundation (English-only)

**Scope:** core

## Added
- `network.vonix.guardian.core.i18n.Messages` — central `get(String key, Object... args)` lookup backed by `java.util.ResourceBundle("lang.en_us")` with `MessageFormat` interpolation. Safe-fallback contract: missing key → returns key itself, malformed pattern → returns raw template. Never throws.
- `core/src/main/resources/lang/en_us.properties` — 29 English message keys covering the entire `QueryParser` player-error surface plus a handful of `/vg status` section headers.
- `core/src/main/resources/lang/README.md` — translator instructions (ISO 639-1 + 3166-1 filenames, `MessageFormat` escaping rules, contribution channel = Vonix Discord `#guardian-i18n`).
- `MessagesTest` — 5 contract tests: known-key resolution, missing-key fallback, positional interpolation, extra-args tolerance, and a smoke check that every extracted key resolves.

## Changed
- `QueryParser` — all 24 player-facing `throw bad(tok, "...")` sites now route through `Messages.get(...)` keys. Wire messages read identically to pre-change output (the parser tests that `hasMessageContaining(...)` continue to pass unchanged).

## Non-goals (deferred to v1.3.0)
- Cell-side strings (Fabric/Paper chat feedback) — this wave stays in core.
- Locale selection / config wiring — `Messages` always reads `en_us`.
- Log strings — operators read English logs; not extracted.

`./gradlew -PbuildProfile=coreonly :core:test` → **526 pass**, 1 pre-existing failure (`VonixGuardianAPITest.apiVersion_and_testAPI_are_stable` expects `"1.1.7"` but base commit `8238c4c` bumped `PLUGIN_VERSION` to `"1.2.0"` — unrelated to this branch).
